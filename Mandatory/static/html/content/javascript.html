Javascript

Short introduction to basic Javascript functions, methods and objects

Functions

The definition of a function is that is is a part of a program made to perform a specific task.

Javascript Functions supports many variations. This introduction will only present some syntaxes & simple examples.
Some of the Javascript variations are:

  • - Anonymous functons
  • - Inner functions
  • - Callback functions
  • - Etc.


Syntax:

When declaring functions, the 'function' keyword is used and followed by a name.

                
    function anyName(parameters) {
        statements;
    };

Using an Arrow Function Expression is a more efficient way to write function expressions. An advantages is shorter syntax: 

        
    function(parameter) => {
        statement; 
    });

A simple example of a Javascript Arrow Function: 

            
    function someEquation(a1, a2, a3){
        return a1 * a3 - a2;
    }

The same example but with Arrow Function: 

    
    someEquation (a1, a2, a3) => {
        return a1 * a3 - a2; 
    });

Variables

Short introduction to Javascript Variables.

In Javascript & all other programming languages, variables are basically containers used to store values inside. These containers can then be refered to by the name given to them in the declaration process.


const brandName = "Jaguar";         //const is a constant variable 
var carModel = "MkII 3.8 Litre";    //var is a Globalscope variable
let manufacturingDate = "1959";     //let is a BlockScope variable

If you wish to create variables, you must declare a variable & then assign it a value. Unassigned variables are by default initialized with undefined.
In the example above, there are declared three variables with different values. In order to create variables, you must use one of the three keywords: const, var or let.

Thumb-rule of Javascript keywords:
  • Const:
    • Use const keyword if the variable is to be changed
  • Let:
    • Use let keyword if the variable is not to be changed
  • Var:
    • Use var keyword if global scope is needed. (If possible - do avoid it)

Scope

Scopes defines where in the code a certain variables & functions are accesible inside the program. There are two kinds of scopes in Javascript:

Global Scope:
A variable is in the global scope if it appears outside a function. In other words, the particalar variable is accesible outside the functions.

Local Scope:
A variable is in the local scope if it appears inside a function. In other words, the particular variable is only accesible inside that function.

Hoisting

Hoising is when Javascript reads the code, it will then move all function and variable declaration to the top of the current scope. In other words, it hoists/rises the code.
Cosnider the following example: 

 
console.log('brand: ', brand)
console.log('type: ', type)

var brand = 'MTB'
var type = 'Mountain Bike'
}

Due to hoisting, the code will actually run & and the console.log will give us 'undefined' as expected.
The program in the interpreter after hoisting:

 
var brand = 'MTB';
var type = 'Mountain Bike';

console.log('brand: ', brand) //undefined
console.log('type: ', type)   //Undefined

Data Types

There are many different kinds of data types in Javascript. Primitive Data Types are called 'primitive' because they are stored by value.
Here are some Primitive Data Types:

  • String
    • Text
  • Number
    • Both float and int values
  • BigInt
    • Whole numbers between -2^53 & 2^53
  • Boolean
    • True & False
  • Undefined
    • Unassigned value
  • Null
    • A special value that represents 'nothing'
  • Symbol
    • Unique identifier

Unlike primitive data types, the complex data types are stored by reference.
Some of the non-primitive data types of Javascript:

  • Object
    • Collection of data and more complex entities
  • Arrays
    • Group of values
  • RegEx
    • Regular Expressions

Objects

Short introduction to Javascript Objects.

In contrast to variables, ojects are used to store keyed collections of different data types (instead of just one).
There are a couple of ways of creating objects in Javascript.
The syntax looks like:

            
    let nameOfObejct = {
        key: value, 
        key: value
    }
    nameOfObejct.key;

An example:

            
    const dog = {          //The object
        race: 'Shiba Inu', //The key 'race', which stores the string value: 'Shiba Inu'
        age: 8,            //The key 'age', which stores the number: 8
        colour: 'brown'    // -||- 
    }
    console.log(dog.race)  //logs the dog's race
    console.log(dog)       //logs all info about the dog

    or 

    console.log(dog['race']);
    console.log(dog['colour']);

As seen in the example above, one of the ways to extract methods or properties from an object is to use the dot notations.
The other way is to use bracket notation, which is also presented in the example.

Objects are not only limited to storing primitive data types. They can also have functions as values.
Here is an example:

            
    const person = {          
        name: 'Gabriel', 
        phoneNumber: '12345678', 
        age: 89, 
        const: updateAge(){
            return ++person.age; 
        }
    }

    console.updateAge();  //runs the updateAge method within the object
    console.log(age);    //90

When the updateAge funcion is called, the age will change to 90.
