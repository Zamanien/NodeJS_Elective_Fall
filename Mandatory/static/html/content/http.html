HTTP

Short introduction to HTTP.

HTTP stands for Hyper Text Transfer Protocol. The HTTP protocol is used across countless web pages on the World Wide Web. The purpose of HTTP is to achieve communication between server & client on the WWW. This is done through HTTP request methods.

HTTP is stateless. This means that each time a connection is established, there are no memories of past communication.
This is also why, sites like Facebook uses Cookies (to remember your login & log your credentials - information).

HTTP Request methods

  • GET method
    • The GET method is used everytime a webpage is loaded. The client (often browsers) send a GET request to the website (web-server) in which the client asks for the relevant data in order to load a specific website.
      It should be noted that the client sends a request for a representation of the resource. The GET method only retrieves data and is also defined as a safe method.
  • PUT methods
    • The PUT method is used to update specific resource / resources. If the request points towards an existing resource, then that resource should be updated with the new data.
      If the request points at a non-existing resource, then it will create the resource. The PUT method is defined as idempotent.
  • POST method
    • The POST method is used to send. When logging into a website, the client (browser) sends a POST request as the user has filled out the login formular. Like the PUT method, the POST method is also considered as idempotent.
  • DELETE method
    • As the name hints, the DELETE request will simply delete the specified resource.

Besides the ones listed above, there are also: HEAD, OPTIONS, TRACE, CONNECT & PATCH, which we won't cover as they are not relevant for us.

Scenario:

1. User searches for an article on [Wikipedia.com](http://wikipedia.com). 2. The browser sends a GET request to the Wikipedia Web-server, which will - if it exists - find the correct response in the correct format. Often just HTML file & pictures. 3. The Web-server returns the correct files and information, which is called a HTTP Response 4. The browser interprets and displays the response to the Client.

HTTP Status Codes

The status codes are the response from the server. One of the most recognisable status code is 404.
The status codes:

  • 100 - 199 | Informal Response
  • 200 - 299 | Succesful Response
  • 300 - 399 | Redirects
  • 400 - 499 | Client Errors
  • 500 - 599 | Server Errors
