NPM

Short introduction to NPM and Frameworks, Packages and Modules.

NPM stands for Node Package Manager. NPM is a tool used to borrow modules, packages and frameworks in NodeJS. Currently, the most used NodeJS framework is Express. NPM is open source & flexible but it is also reliable, which is why it is are used by big companies, such as Reddit & Stack.
There are basically three parts to NPM:

  • The Website
    • The NPM website can be used to discover new or find exisiting packages. Let's say that you search for Express on the NPM website.
      The site will provide many neat things about Express, such as version, documentation, examples, dependencies etc.

  • The Registry
    • The NPM Registry is a database that consists of Javascript code. The code is ready to be downloaded through the NPM client.

  • The Command Line
    • The NPM Command Line comes with NodeJS when it's installed on a device.
      This is where the developer runs the NPM commands to download & install packages from the NPM Registry. Some of the NPM commands can be seen in the table below.

NPM Commands:
NPM Init NPM Install NPM Start

The npm init command will initialize a package.json file in the profect folder. This is typically the first step before installing Node Dependencies. The package.json file will look like the following:

  • "name": "nodeProject",
  • "version": "0.0.1",
  • "description": "Project description",
  • "main": "app.js", //Main js file. Usually called 'app.js' due to convention.
  • "author": "My Name",
  • "license": "ISC" //License of your choosing
  • "dependencies": {
  • "express": "^4.17.1"
  • }

Npm install | with arguemnts:
Running the 'npm install (argument)' or 'npm i (argument)' will download the package and relevant dependencies, which are saved to the - already existing - 'node_modules' folder. Npm install with arguments will not create a new 'node_modules'.

Npm install | w/o flags:
Running the 'npm install' or 'npm i' will download the modules, which are defined in the 'package.json' file. Specifically in the depenency section.

Running 'npm start "scriptName"' will launch the script.

Packages vs. Modules

What is the difference between a package and a module in NodeJS?:

Packages are one or more modules packaged together.
Modules are libraries for NodeJS.

Example: 

Animal         --> Name of package
    - Dog       ->
    - Chicken   ->  Modules inside Package: Animal  
    - Horse     ->

Every file in a Node application is considered a package and should have a 'package.json' file.


Some Frameworks

Express

Express is a Nodejs framework often used for web applications & provides a lot of neat features.

    Installation

    Before Express can be installed, NodeJS needs to be installed on the device. One way to install Express is to use NPM.
    1. Open the terminal in the editor.
    2. Write: 

      npm i express
    Webserver example with Express:
    1. Create an 'app.js' file in the directory (app.js is used by convention, but can be called anything)
    2. Type the following inside the 'app.js' file: 
                        
  const express = require('express');
  const app = express();
  const port = process.env.PORT || 9999;

    ...

  app.listen(port, (error) => {
        if(error) {
            console.log("Port Error");
        }
      console.log("up and running on port:", Number(port);   
    });
    3. Lastly, launch the app.js file: 

      node app.js

By running the app.js file with node, the web server will launch on port 9999. The site can be accessed through: localhost:3000 but nothing will be displayed because we have not asked the application to respond with anything specific yet.
The application can in this example return an HTML file. This can be done by routing some endpoints in the 'app.js' file. These endpoints are provided by Express.

    Syntax:

    
    app.HTTP-method("/URL-Path", (Handler)) {
        return respone + "File-Path"
    }

    Example:

        
    app.get("/npm", (req, res) => {
        res.sendFile(__dirname + "/static/html/content/npm.html");
    });

The example above is actually how the endpoint for this specific page looks like.
We set the URL-path to: '/npm' and we also set the reponse to be the '../npm.html' file, so that our web-server responds with a representation of that specific resource when a client makes a GET-request to that URL.
Endpoints are our application's URI's.

Nodemon

Nodemon is a NodeJS tool & can be downloaded through the NPM registry (or through cloning with git). It is an extreme useful tool when it comes to web development as it automatically restarts the Node application everytime a change is made in the backend.
This may not sound impressive but imagine manually restarting your application everytime you make a change. By running the application with Nodemon, the other issue is seen as a thing of the past.

Install & Use

Nodemon is installed like many other tools in NodeJS.

  • 1. Open the terminal.
  • 2. Type: 'npm i nodemon'
    • It is possible to add the -g flag, which will install the tool in the global scope.
  • 3. Now you have installed needed packages & dependencies.
    You can simply type the following in your terminal to continously run your application: 'nodemon app.js'.
